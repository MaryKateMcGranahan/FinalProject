History of Steganography

Not all secret messages have to be written out in jumbled up letters. Steganography is the art of hiding messages amongst non secret text or data. It began in ancient Greece in a couple of forms, one of them an ancient Greek named Demaratus came up with when wrote a message warning of a Spartan invasion on the wooden backing of a wax tablet and hid it by filling it with wax so the tablet looked blank. Today, images can hold messages in their digital code. Every pixel in a picture corresponds to a hexadecimal number. Here is a picture of a puppy next to its hexadecimal code:

Hexadecimals

Hexadecimal numbers are base 16. We need to convert them into binary in order to read the secret message in them. As you may notice, hexadecimals have letters in them in addition to their numbers. Here is how hex counts: 0, 1, 2... 9, A=10, B=11, C=12, D=13, E=14, F=15. Each hexadecimal digit corresponds to four bits of binary, so you can convert each digit individually. Lets say we have the hex 1F. We would use the methods we learned on the last page to convert 1 into binary= 0001 and 15 into binary= 1111. So teh full conversion is 00011111.

Hiding the Message in Binary

In order to hide a message in binary, you need to match the binary of your message to the last binary digits of some pixels in the picture. You do this by adding 1, taking away 1, or leaving the last bit of the binary the same. For example, if I had the hex code of AC09 8A... and I wanted to hide the message hey, I would convert hey into its binary string based on its decimal value in the ascii chart= 01101000 01100101 01111001. I would then convert the first hex pixel, AD, into binary= 10101101. The first bit of the string we are hiding is 0, the last bit of the pixel we are starting at is 1, so we change the 10101101 of AD into 10101100, making it AC. These minor changes to the pixels will not affect the actual image. It is also helpful to know the start index of the hidden message (the number of the first changed pixel), or else you could be searching through thousands of extra pixels.
